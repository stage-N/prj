## 7月から「法人印」がクラウドに——何が変わるのか

2026年7月から、会社・法人の代表者が使う **商業登記電子証明書**（紙の印鑑証明書に相当）に、新しい使い方が加わりました。名前は **商業登記リモート署名** です。

従来は、署名鍵をパソコン上のファイルとして保管し、そのPCがないと署名できませんでした。出張先や自宅以外では、e-Taxや登記のオンライン申請に署名を付けられない——1人法人の私も、証明書ファイルを「会社用PC」に縛られている感覚がありました。

リモート署名では、署名鍵を法務省のクラウド上で管理し、**スマートフォンのGビズIDアプリ**に表示される4桁の認可コードで承認します。紙でいえば「実印をクラウド金庫に預け、押印のたびにスマホで許可する」イメージに近いです。

重要なのは、**既存の証明書ファイルをそのままリモート署名に使えない**点です。リモート署名を使うには、新しい手順で証明書を取得し直す必要があります。一方、今お持ちのファイル形式の証明書は、有効期限までは従来どおりローカル署名として使えます（法務省 FAQ より）。

## 紙・ローカル・リモート——3つの世界の住み分け

読者向けに、私がメモしている対応表です。

| 方式 | 鍵の場所 | GビズID | 代表的な用途 |
|------|----------|---------|--------------|
| リモート署名（2026年7月〜） | 法務省クラウド | **必須** | 外出先からの行政申請・Acrobat署名 |
| ローカル署名（ファイル形式） | 自分のPC | 不要 | 従来どおり。2030年頃まで民間ソフトでも取得可 |
| 立会人型（クラウドサイン等） | 事業者側 | 不要 | 契約書のやり取り。印鑑証明と同等の効力は別 |

CloudSignのような立会人型は手軽ですが、**代表者本人の印鑑証明に相当する署名**が必要な行政手続きには使えません。e-Taxや登記・供託のオンライン申請では、商業登記電子証明書が求められる場面が残ります。

デジタル庁の資料では、GビズID利用者は100万を超える一方、商業登記電子証明書を使う企業は2023年時点で約6万社（全法人の約1.6%）とされています。制度は2000年からあるのに普及が止まっていた背景のひとつが、「PCとファイル管理」だった、と私は理解しています。

## 数字で見る——費用と市場の目安

証明書の発行手数料は、2025年4月に引き下げられ、**最短1ヶ月500円**から選べます（法務省公表）。

| 証明期間 | 手数料 |
|----------|--------|
| 1ヶ月 | 500円 |
| 3ヶ月 | 1,100円 |
| 6ヶ月 | 2,000円 |
| 12ヶ月 | 3,800円 |
| 27ヶ月 | 8,300円 |

GビズIDアカウント自体は無料です。リモート署名ドライバソフト（Windows向け）も法務省提供で無料です。コストの主役は「証明期間をどれだけ取るか」と、登記所への発行申請の手間です。

1人法人で、e-Taxを年に数回、登記変更をたまに——という使い方なら、まず1〜3ヶ月の短期で試し、運用が固まってから12ヶ月に伸ばす、という段階も現実的です。代表者変更など登記事項に変更が入ると証明書は失効し、手数料の払戻しはありません。期限カレンダーとセットで考えると安心です。

## 全体の流れ——4フェーズで覚える

公式の手順は細かいですが、大きくは次の4段階です。

**フェーズ0: 事前準備**
- GビズIDアカウント（プライムまたはエントリー）
- スマートフォンにGビズIDアプリをインストール

**フェーズ1: 発行申請の準備（ブラウザ）**
- [商業登記電子認証ポータル](https://portal.cr-rs.moj.go.jp/) にGビズIDでログイン
- 「鍵ペア及び証明書発行申請ファイル作成」で SHINSEI ファイルを生成
- **鍵パスワード**を設定（紛失すると復旧不可——法務省も再確認できないと明記）
- **認可端末**を登録（ポータルのQRをGビズIDアプリで読み取り、スマホと証明書を1対1で紐付け）

**フェーズ2: 登記所へ発行申請（従来と同じ）**
- 窓口・郵送、または申請用総合ソフトによるオンライン申請
- 審査後、**シリアル番号**が通知される

**フェーズ3: 証明書の取得・登録**
- ポータルでシリアル番号と鍵パスワードを入力し、クラウドへ登録

**フェーズ4: 署名**
- 手続き画面でファイルを選択 → PCに4桁の**認可コード**表示 → GビズIDアプリで入力 → 完了

Adobe Acrobat等で署名する場合は、Windows 11に**リモート署名ドライバ**の追加インストールが必要です。MacやiPhoneだけではドライバ方式の署名はできません（ポータル自体の閲覧・申請準備はMacでも可）。

## つまずきやすい用語——5つだけ押さえる

私が最初に混乱したのは、似た名前のパスワードとコードの違いです。

| 用語 | 役割 |
|------|------|
| **鍵パスワード** | 証明書取得・署名時にPC側で入力。自分で決め、忘れたら終わり |
| **認可端末** | 署名を許可するスマホ（1証明書につき1台のみ） |
| **認可コード** | 署名のたびにPC画面に出る4桁。アプリに入力して承認 |
| **シリアル番号** | 登記所が交付する番号。証明書ダウンロードに必要 |
| **SHINSEI** | 登記所へ提出する発行申請ファイル名 |

GビズIDの **メンバー** アカウントは、原則として発行準備ができません。会社の支配人であるメンバーだけ、プライムの承認を経て例外になります。1人法人の代表が自分でプライムを持っている場合は、ここを気にしなくてよいケースがほとんどです。

## 1人法人代表のチェックリスト——今日からできること

税理士に聞く前に、自分で整理できる範囲として、私は次の順番をメモしています。

1. **GビズIDプライムがあるか確認** — なければマイナンバーカードで即日昇格も可能（デジタル庁説明会資料）
2. **既存の証明書の有効期限と形式を確認** — ファイル形式のまま期限まで使うか、リモートへ移行するか決める
3. **主な用途を1つ書く** — e-Tax / 登記オンライン / 契約書署名。対応方式が手続きごとに異なるため
4. **使うPCのOSを確認** — Acrobat署名ならWindows 11が必要
5. **鍵パスワードの保管場所を決める** — パスワードマネージャー等。紛失は自己責任

リモート署名は「税務の中身」を変える制度ではありません。でも、e-Taxや法人届出をオンラインで回す代表にとっては、**署名の場所と端末の制約が緩む**のは実務上の大きな変化です。期限（法人税申告・中間申告・登記変更）とセットで考えると、7〜9月にセットアップを済ませておく動機づけにもなります。

## 私が道具として持っていること——ZeiCalとの関係

上の整理は、STAGENで1人法人を運営する私自身のメモでもあります。税務の**判断**は税理士、**期限とルーティン**は自分で持つ——という住み分けのなかで、電子署名は「届出・申告の直前に毎回つまずくポイント」でした。

**ZeiCal** は、法人税・消費税などの期限を1カ所に集めるカレンダーです。申告代行や税務相談は行いません。リモート署名の取得自体もアプリの機能ではありません。ただ、代表者が「いつまでに何を用意するか」を先に見えていると、証明書の更新月と申告月が重なったときの慌て方が変わります。
